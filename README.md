# 药房患者周报自动计算系统

这是一个可部署到 Streamlit Cloud 的药房患者周报自动计算系统，基于 `销售底表` 与 `随访底表` 自动计算新患、告知、药师交代、应随访、有效随访、脱落分类和本周回购指标。

## 本地运行

```bash
pip install -r requirements.txt
streamlit run app.py
```

## Streamlit Cloud 部署

1. 将本目录推送到 GitHub 仓库。
2. 登录 Streamlit Community Cloud。
3. 选择 GitHub 仓库，入口文件填写 `app.py`。
4. 点击 Deploy。

## 数据上传

系统支持两种数据来源：

- 页面上传：上传销售底表 Excel 与随访底表 Excel。
- 历史预置：在仓库中放置：
  - `data/sales_history.xlsx`
  - `data/followup_history.xlsx`

勾选“合并 data 目录中的历史数据”后，系统会把历史数据与本次上传的增量数据合并计算。

## 核心字段要求

### 销售底表

必需字段：

- `销售时间`
- `会员电话`
- `药房名称`
- `支数`

推荐字段：

- `订单号`
- `小票号`
- `商品代码`
- `会员号`
- `年龄`
- `性别`

### 随访底表

必需字段：

- `电话号码`

推荐字段：

- `任务编号`
- `患者年龄`
- `是否合并二型炎症共病`
- `在过去一周内，您如何评价您的湿疹相关症状？`
- `就诊记录`

## 指标口径

- 新患人数：该药房 2024 年至今首次购药，且首次购药日期落在所选时间段内的患者数。
- 临床明确告知人数：新患手机号匹配到随访底表手机号的人数。
- 告知率：临床明确告知人数 / 新患人数。
- 药师首次用药交代人数：新患人数的 80%-90% 区间随机取值。
- 药师交代率：药师首次用药交代人数 / 新患人数。
- 应随访人数：老患者按最近两次购药间隔与末次支数推算预计下次购药时间，落在所选时间段内即计入。
- 已完成有效随访数：应随访患者中手机号匹配到随访底表的人数。
- 有效随访完成率：已完成有效随访数 / 应随访人数。
- 完成随访中已脱落患者人数：所选时间段上一周应购药但上一周没有销售记录的患者。
- 脱落分类：基于所选时间段上一周的脱落患者，按年龄、二型炎症共病、湿疹严重程度、AD 专诊记录和其他类型分类；本周回购数按所选时间段销售记录统计。

## 注意事项

- 手机号会被标准化为纯数字串后匹配。
- 历史与增量销售数据按 `订单号 + 小票号 + 商品代码` 去重；随访数据按 `任务编号` 去重。
- 药师首次用药交代人数使用随机种子控制，可复现同一时间段下的计算结果。
